# Audio assets — sourcing + licensing

Spec §11 lists 10 sounds. The audio system (`apps/webapp/src/audio/`) was
wired in Phase 5 and shipped silent for a long time — no files existed, so
every `audio.play()` was a no-op on every device.

**As of 2026-06-01 the set is filled by procedural synthesis.** All ten
cues are generated by [`scripts/generate_sounds.py`](../scripts/generate_sounds.py)
— pure-stdlib additive/subtractive synthesis tuned around C major, encoded
to mono MP3 via ffmpeg. They are **100% original, no third-party license**
(no attribution needed, safe for commercial use). Regenerate / tweak:

```bash
python3 scripts/generate_sounds.py   # writes apps/webapp/public/sounds/*.mp3
```

To swap in higher-fidelity recordings later, drop replacements with the
same names into `apps/webapp/public/sounds/`. If you do, use only
royalty-free, license-clean sources and log them below:

- [Freesound](https://freesound.org) — filter for CC0
- [Pixabay Sound Effects](https://pixabay.com/sound-effects/) — Pixabay license, free to use
- [Mixkit](https://mixkit.co/free-sound-effects/) — free for commercial use
- [Zapsplat](https://www.zapsplat.com) — free with attribution / no-attribution paid

## Why the system is silent without assets

`AudioManager.play()` (in `apps/webapp/src/audio/index.ts`) creates each
`Howl` lazily on first use and registers `onloaderror` + `onplayerror`
handlers that swallow errors. The rest of the codebase remains
indifferent to whether sound files are present.
