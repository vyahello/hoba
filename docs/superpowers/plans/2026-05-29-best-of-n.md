# Best-of-N Implementation Plan

> **For agentic workers:** execute inline (lean). Steps use `- [ ]` tracking.

**Goal:** Host-chosen `spin_count` ∈ {1,3,5,7}; server runs N spins, most-frequent segment wins (ties → tie-break spins among leaders); client animates fast sub-spins + live tally + dramatic final. Classic + solo this slice.

## Task 2: run_spin_series helper (pure)
- `services/spins.py`: new `run_spin_series(*, segment_count, n, starting_angle_deg, weights, seeds) -> list[SpinResult]` — chains angle forward across n compute_spin calls. Plus `tally_winner(results, segment_ids) -> (winner_index_into_segments, tally_dict, tie_break_results)`: count hits; on tie run extra spins restricted to tied leaders (recompute over full wheel, ignore non-leader landings) until single leader or cap 20 → random leader.
- Keep it pure (seeds passed in; caller uses secrets). Actually simpler: helper takes a `rng_seed_fn` or just does `secrets.randbits(31)` internally — acceptable since compute_spin already seeded server-side. Use internal secrets.
- Signature: `def run_spin_series(segments: list[Segment], n: int, starting_angle_deg: float, weights: list[float] | None) -> tuple[list[dict], Segment, int]` returning (series_entries, winner_segment, tie_break_count). Each entry: `{segment_id, final_angle_deg, duration_ms, seed}`.
- Tests (`tests/services/test_spins.py`): n=1 → 1 entry, winner=that; n=3 deterministic via monkeypatched compute? Instead assert: len(series)==n; winner has max tally; angles strictly increasing. Force a tie scenario with 2 segments + seeded determinism is hard — test tie-break by asserting: with n=2 and a forced tie (mock), tie_break_count>0 and a single winner emerges. Use a small wrapper test that drives many runs and checks winner always in segments + series length == n (+ tie_break entries appended).
- Commit `feat(api): run_spin_series helper (N spins + tie-break)`.

## Task 3: trigger_spin uses series for classic
- In `trigger_spin`, after computing `spin_segments`/weights/starting_angle:
  `effective_n = room.spin_count if room.game_mode == "classic" else 1`.
- If `effective_n == 1`: keep current single-spin path (unchanged) BUT also store `series` with the single entry for client uniformity.
- Else: `series, winner_segment, tie_break = run_spin_series(spin_segments, effective_n, starting_angle, weights)`; `winning_segment = winner_segment`; `final_angle_deg`/`duration_ms` = last series entry; `mode_state_snapshot` gets `{"series": series, "winner_segment_id": winner.id, "tie_break_count": tie_break, "mode_effects": {}}` (+ existing keys for other modes preserved on the n==1 path).
- Tests: classic spin_count=3 → spin.mode_state_snapshot["series"] len>=3, winner_segment_id set; elimination with spin_count=3 still len(series)==1 (forced N=1).
- Commit `feat(api): trigger_spin runs best-of-N for classic`.
